# 퀵서베이 프로젝트 — 기획팀과의 대화 기록 (2026-06-22)

기획팀 PM과 나눈 일정/방향성 확인 대화. 추후 작업 방향 참고용.

## 일정 관련

- 기본 구조를 포함한 기획(안): **다음주 초까지** 공유 예정
- 상세 기획(안) 일정: **개발 범위 확정 후** 전달
- 목표 오픈일: 상세 개발 범위가 아직 확정되지 않아 미정, 최대한 서둘러 공유 예정
- 본 프로젝트는 **네이트 단독 진행이 아님** — 외부 제휴업체 상황에 따라 일정/개발 범위가 변동될 수 있음 (현재 제휴 기업 컨택 중)

## 개발 방향성 확인 (중요)

- **자체 설문 빌더(오픈서베이 수준의 고도화 기능)를 만들 계획은 없음** — PM이 명확히 확인
- 방향은 "기존에 갖고 있던 기능(레거시 뉴스 폴)에서 좀 더 고도화"하는 정도
- 즉, 지금 레거시 뉴스 폴을 대체/신규 플랫폼 구축이 아니라 **레거시 고도화** 쪽이 1차 방향
- 신규 개발 / 기존 기능 보완 사항은 이미 분류되어 있고, 운영 이슈 해결되는 대로 공유 예정

## 이 저장소(quicksurvey-fe)와의 관계

- `quicksurvey/`, `quicksurvey-builder/`는 FE 선행개발 — PM 측 기획/개발 범위 확정과 무관하게 미리 만들어보는 프로토타입
- 위 방향성 확인에 따르면 실제 레거시 적용 시에는 "설문 빌더"라는 별도 도구보다 **레거시 폴 기능의 고도화** 형태로 들어갈 가능성이 높음 — 다만 이 저장소의 선행개발 자체는 자체 플랫폼(3·4단계) 방향 검증용으로 계속 의미가 있음
